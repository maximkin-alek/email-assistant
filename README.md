# Email Assistant (MVP)

## Возможности

- **Синхронизация почты**:
  - IMAP (Яндекс/Mail.ru/и др.)
  - Gmail через OAuth
  - синхронизация “всё” и синхронизация конкретного ящика
  - авто‑синк в фоне (по расписанию)
- **AI‑обработка писем**:
  - короткие summary/explanation (ограничение по длине)
  - устойчивость к “ломаным” ответам моделей и таймаутам
- **AI‑саммари по дню**:
  - блок “Сводка дня” на главной странице (темы/цепочки, требует внимания)
  - кнопка “AI‑выжимка дня” — делает общий дайджест по сегодняшним письмам
- **Управление ящиками в UI**:
  - добавление IMAP и Gmail
  - включить/выключить
  - удалить (вместе с письмами/вложениями ящика)
  - импорт стандартных IMAP‑кредов из `.env` (для миграции/удобства)
- **Интерфейс**:
  - вкладки ящиков и визуальные цвета
  - просмотр письма, вложения
  - архив/разархивирование
  - подсказки в настройках: `?` рядом с сервисными кнопками

## Справка

### Как “правильно” пользоваться приложением (MVP)

- **Шаг 1. Подключи ящики**: `Настройки` → добавь IMAP и/или Gmail.
- **Шаг 2. Синхронизируй**: во `Входящих` нажми **“Синхронизировать”** (или синхронизацию конкретного ящика в настройках).
- **Шаг 3. Разбери день**:
  - сверху есть **“Сводка дня”**: темы/цепочки, подсказки “требует внимания”, массовые действия
  - кнопка **“AI‑выжимка дня”** делает текстовый дайджест по сегодняшним письмам
- **Шаг 4. Действия по письмам**: архив, прочитано, категория (важно/рассылка/спам?).

### Что означают категории

- **Важно**: письмо, которое вероятнее всего требует внимания/действий.
- **Рассылка**: новости/маркетинг/уведомления, обычно можно читать пачкой и архивировать.
- **Спам?**: кандидаты на очистку (пометка эвристическая; перед удалением лучше просмотреть).

### Архив vs Входящие

- **Входящие** показывают неархивированные письма.
- Если “письма пропали” — часто они **в архиве** или включён фильтр/поиск/группа.

## Технологии

- **Backend**: Python 3.12, FastAPI
- **Шаблоны/UI**: Jinja2 + CSS/JS (без отдельного фронтенд‑фреймворка)
- **База данных**: PostgreSQL (SQLAlchemy ORM)
- **Очередь задач/фоновые джобы**: Redis + RQ
- **AI**: OpenAI‑compatible API через `httpx`
- **Шифрование секретов**: `cryptography.Fernet` (в БД)
- **Контейнеризация**: Docker Compose (`web`, `worker`, `db`, `redis`)

## Быстрый старт (Docker Compose)

⚠️ **Важно**: файл `.env` содержит секреты. Он **не должен** попадать в git (в проекте это уже добавлено в `.gitignore`).

1) Сгенерируй ключ шифрования (один раз):

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

2) Создай `.env` (можно скопировать из `.env.example`) и укажи минимум:

- `APP_ENCRYPTION_KEY=...`
- **AI** (RouterAI или другой OpenAI‑compatible) — см. ниже
- **Gmail OAuth** (если нужен Gmail) — см. ниже

3) Запусти:

```bash
docker compose up --build
```

Открой `http://localhost:8000`.

## Подключение AI (рекомендуется: RouterAI)

RouterAI даёт **OpenAI‑совместимый** `chat/completions` API и оплату в рублях.

В `.env`:

- `AI_BASE_URL=https://routerai.ru/api/v1`
- `AI_API_KEY=...`
- `AI_MODEL=deepseek/deepseek-v3.2` (или другую доступную модель)

## Резервный AI: Cursor через локальный proxy

Ключ из `https://cursor.com/dashboard/cloud-agents` относится к **Cursor Cloud Agents API** и не является OpenAI‑совместимым `chat/completions`.

Чтобы использовать Cursor в этом проекте, нужен локальный **OpenAI‑compatible proxy** для Cursor CLI (например `cursor-api-proxy`).

Минимальная схема на Windows:

1) Установи Node.js 18+.
2) Установи Cursor agent CLI и залогинься:

```bash
curl https://cursor.com/install -fsS | bash
agent login
agent --list-models
```

3) Запусти proxy:

```bash
npx cursor-api-proxy
```

4) В `.env` для контейнеров укажи:

- `AI_BASE_URL=http://host.docker.internal:8765/v1`
- `AI_API_KEY=unused` (или `CURSOR_BRIDGE_API_KEY`, если включал авторизацию в proxy)
- `AI_MODEL=...` (лучше фиксировать конкретную модель)

## Подключение почты

### IMAP (Яндекс / Mail.ru / другие)

1) Открой **Настройки → “Добавить IMAP ящик”** и введи:

- host (например `imap.yandex.ru`)
- порт (обычно `993`)
- логин (обычно полный email)
- пароль (часто нужен **пароль приложения**, если включён 2FA)
- папка (обычно `INBOX`)

2) Нажми **“Синхронизировать”** рядом с ящиком или **“Синхронизировать всё”**.

#### Что такое “пароль приложения”

Если в почте включена двухфакторная защита (2FA), обычный пароль может не подойти для IMAP.
Тогда в настройках аккаунта почты создаётся **отдельный пароль приложения** — специальный пароль для “старых” протоколов (IMAP/SMTP) и сторонних приложений.

- Его можно **создать/отозвать** в настройках безопасности аккаунта провайдера (Яндекс/Mail.ru/и т.д.).
- Он обычно выглядит как **отдельная строка/код** и вводится в поле “Пароль / пароль приложения”.

#### Если включён VPN/антивирус и IMAP падает по сертификату

Некоторые VPN/антивирусы подменяют TLS‑сертификаты (MITM). В этом случае может быть ошибка вида `CERTIFICATE_VERIFY_FAILED`.

- В настройках ящика можно выключить **TLS проверку сертификата** (кнопка `TLS verify on/off`).

⚠️ Это снижает безопасность соединения. Лучше отключить VPN/перехват, если возможно.

#### Импорт IMAP из `.env` (миграция/удобство)

Если у тебя уже есть переменные `YANDEX_IMAP_*` / `MAILRU_IMAP_*` в `.env`, можно нажать в настройках **“Импортировать ящики из .env”** — ящики будут созданы/обновлены в БД и сразу поставятся в очередь на синк.

### Gmail (OAuth)

1) В Google Cloud Console создай OAuth Client (Web application).
2) В Authorized redirect URI укажи:

- `http://localhost:8000/oauth2/google/callback`

3) В `.env`:

- `GMAIL_OAUTH_CLIENT_ID=...`
- `GMAIL_OAUTH_CLIENT_SECRET=...`
- `GMAIL_OAUTH_REDIRECT_URI=http://localhost:8000/oauth2/google/callback`

4) В UI нажми **“Добавить Gmail”** и пройди OAuth.

> Примечание: приложение старается не создавать дублей Gmail‑ящиков при повторных нажатиях/переподключениях.

## Полезные заметки и типичные проблемы

- **`.env` никогда не коммитим**: он игнорируется. Используй `.env.example` как шаблон.
- **IMAP “handshake timeout”**: обычно проблема сети/фильтрации/неверного host/порта.
- **IMAP “certificate verify failed”**: чаще всего VPN/антивирус подменяет сертификаты — можно временно отключить TLS verify в настройках ящика.
- **Писем не видно во “Входящих”**: проверь вкладку **“Архив”** и сбрось фильтры (в UI есть быстрые кнопки в пустом состоянии списка).
- **Логи**:
  - `docker compose logs -f web`
  - `docker compose logs -f worker`

## Планы развития

- **Авторизация**: логин/пароль или OAuth, чтобы разносить данные по пользователям.
- **Ответы на письма**: черновики и отправка ответов (IMAP/SMTP и Gmail API), шаблоны и тональность.
- **Умные действия**: предложить “что сделать” (архивировать/пометить/создать задачу) и выполнить в один клик.
- **Автоматизация**: правила/триггеры “если … то …” (например, авто‑архив рассылок, авто‑важность по доменам).
- **Интеграции**: календарь/таск‑трекер/мессенджеры, уведомления.
